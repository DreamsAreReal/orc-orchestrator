# ADR-0002 — Конфиг JSON + гейт как bd-задача с сортировкой в конец

## Статус
Принято 2026-07-15

## Контекст
Два решения-трейдофа: (1) формат конфига калибровок; (2) как реализовать пользовательский гейт ТЗ задачи при выборе пользователя «ждать живьём» на 1-воркерной 8GB-машине.

## Рассмотренные варианты
| Вариант | Плюсы | Минусы | Замеры из спайка |
|---|---|---|---|
| Конфиг TOML | читаемее для человека | tomllib нет в python 3.9.6; внешний tomli = зависимость | spike: `import tomllib` → ModuleNotFoundError на 3.9.6 |
| Конфиг JSON | stdlib json; парсится и jq (bash-обёртки) | менее уютный синтаксис, без комментариев | spike: `import json` OK; `/usr/bin/jq` есть |
| Гейт: живое ожидание, задача в начале | проще | один гейт блокирует всю смену немедленно | коллизия из R-research р2 (1 воркер) |
| Гейт: живое ожидание, гейтовые задачи в КОНЕЦ очереди | автономные проходят первыми; блокировка только когда автономных не осталось | смена всё же встанет на последнем гейте (принятый пользователем trade-off) | выбор пользователя на гейте ТЗ 2026-07-15 |

## Решение
(1) **Конфиг JSON** (`~/.orc/config.json`) — tomllib недоступен в 3.9.6, JSON в stdlib и парсится jq в bash-обёртках; zero-dependency (ADR-0001). (2) **Гейт = блокирующая bd-задача** «approve-ТЗ-<слаг>»; воркер, дойдя до гейта, шлёт macOS-уведомление и **ждёт живьём** (выбор пользователя); диспетчер сортирует гейтовые/ждущие-человека задачи в КОНЕЦ ready-очереди, чтобы автономные задачи прошли, пока пользователь не за компом. Блокировка смены на живом гейте — сознательно принятый пользователем trade-off ради простоты и предсказуемости.

## Последствия
- config.json без комментариев → документировать поля в README.
- Смена с гейтовой задачей раньше конца может простоять; смягчается сортировкой, полностью не устраняется (документировать в газете как «ждёт тебя — смена на паузе»).
- Апгрейд к «завершать сессию на гейте» (освобождать слот) остаётся возможным в P1 без переделки очереди (та же bd-семантика).
