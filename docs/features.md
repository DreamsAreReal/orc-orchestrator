# Фиче-лист — Автономный таск-контур «orc»

North Star: утром накидал ~10 задач по проектам — Mac сам довёл их конвейером pipeline до конца без потери качества.

## Дисциплина статусов
- Каждая фича рождается `todo` (= failing). F1 — walking skeleton. Гранулярность: 1 такт/≤3 критерия.
- `self-pass` ставит builder (доказательство: команда→вывод в `docs/evidence/F<N>/`). `verified` ставит оркестратор по вердикту evaluator-а (тест ACT→OBSERVE→PERSIST→ROUND-TRIP).
- `cut` — фича вырезана целиком. Смоук на старте сессии builder-а: исполняемые проверки скелета + золотого пути.
- CLI-продукт: «глазами» = вывод команд, не скриншоты.

## Фичи

### F1 — Walking skeleton: сквозная смена из 1 задачи + газета + canary [M1] [золотой путь]
Зачем: весь такт опыта (накидал→смена→газета) тонким срезом; signature (газета+canary) — в скелете, не заглушкой.
Что: `orc add` одну задачу в тестовый проект → `orc start` → canary-предполёт → диспетчер claim→spawn терминала с claude → задача идёт → по завершении `orc status` печатает газету со сводкой и путями.
Приёмка:
- [ ] `orc add orc-test-1 "создай hello.txt со словом ready"` создаёт bd-задачу, видна в `bd ready --json`
- [ ] canary-предполёт печатает отчёт (bd/auth/ccusage/RAM/spawn-терминала); при подставном фейле — смена не стартует
- [ ] `orc start` спавнит РЕАЛЬНЫЙ терминал с интерактивным claude в папке проекта (не headless), задача выполняется, hello.txt появляется
- [ ] `orc status` печатает газету: первая строка = сводка (N done/parked/failed + % окна), ниже пути; первый экран ≤150 слов
Проверка: `bash .verify/e2e-skeleton.sh` (создаёт orc-test-1, гоняет, ассертит hello.txt + формат газеты) + вывод в `docs/evidence/F1/`
Статус: todo

### F3 — Диспетчер-ядро: ready→claim→re-validate→preflight→mutex→spawn [M1] [золотой путь]
Зачем: сердце автономии; project-mutex + порядок = «по умному».
Что: цикл: `bd ready --json` (сорт: гейтовые в конец) → claim → re-validate ТЗ-дельты (продуктовый слой изменился → пометка) → preflight (чистое git-дерево; грязное «не наше» → парковка) → project-mutex (1 задача/репо, 1 воркер глобально) → spawn → регистрация PID в shift.json.
Приёмка:
- [ ] две задачи одного проекта НЕ идут одновременно (project-mutex); интервалы активности не пересекаются
- [ ] грязное git-дерево «не наше» → задача паркуется с причиной, не стартует
- [ ] re-validate: изменённый продуктовый слой после claim → пометка в STATE задачи
Проверка: `python3 -m pytest tests/test_dispatcher.py` (моки bd + git-фикстуры)
Статус: todo

### F4 — Admission + back-pressure (лимиты/RAM) [M2]
Зачем: тратить недоиспользуемый пул безопасно, не сжечь weekly-кап.
Что: перед spawn — `ccusage blocks --active --json` (окно) + free-RAM + детект лимит-строк CLI. Строки session/weekly/Opus → парковка до ресета (парс времени); 429/529 → ретрай без парковки. Бюджет-кап задачи/смены из конфига.
Приёмка:
- [ ] на фикстурах лимит-строк: session→парковка+время ресета, weekly→глубокая парковка, Opus→деградация, 429→ретрай (100% фикстур)
- [ ] допуск: ready≠∅ + окно/RAM ok → spawn ≤60 сек; при нехватке RAM — не спавнит
- [ ] бюджет-кап смены превышен → новые задачи не стартуют, запись в газету
Проверка: `python3 -m pytest tests/test_admission.py` (фикстуры в tests/fixtures/limit-*.txt)
Статус: todo

### F5 — Watchdog + восстановление + lease [M2]
Зачем: безнадзорная надёжность; выход из meltdown/зависаний.
Что: heartbeat из PostToolUse; PreToolUse-маркер «tool-in-flight» отличает работу от тишины. Петля (K=конфиг одинаковых hash) / тишина-без-маркера → внешняя проверка пост-условий (git/артефакты) → kill → свежий рестарт от STATE.md, cap=конфиг → эскалация. Lease TTL: мёртвый PID → задача в ready. kill -9 диспетчера → рестарт подхватывает shift.json без дублей.
Приёмка:
- [ ] синтетическая петля (K одинаковых) и тишина-без-маркера детектятся; живой Bash-вызов ≥2 мин НЕ убивается (0 ложных kill)
- [ ] рестарт только после внешней проверки пост-условий; cap соблюдается → эскалация
- [ ] kill -9 диспетчера посреди смены → рестарт продолжает, 0 дублей/потерь; мёртвый воркер → задача в ready
Проверка: `python3 -m pytest tests/test_watchdog.py` + `bash .verify/kill-restart.sh`
Статус: todo
