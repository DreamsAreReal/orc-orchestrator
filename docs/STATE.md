## Рекап задачи (ЖИВОЙ — обновляется при смене вводных)
Цель: утром пользователь накидывает ~10 задач по разным проектам в очередь; на его Mac (Max x20 от подписки, лимиты недоиспользуются) инстансы Claude Code сами поднимаются, берут задачи и гонят каждую через конвейер pipeline до конца.
Жёсткие требования пользователя: (1) ИНТЕРАКТИВНЫЕ сессии — оркестратор открывает терминал, cd в найденную/созданную папку проекта, запускает `claude "<промпт>"`; модель и bypass уже в глобальном settings.json; НЕ headless-демоны. NB: механика гейта (живое ожидание в AskUserQuestion vs завершение сессии на гейте с резюмом — коллизия с 1 воркером на 8GB) — РАЗВИЛКА, решается пользователем на гейте ТЗ, не синтезом. (2) Мультизадачность в проекте — РАЗДЕЛЯТЬ: продуктовый слой docs/ (design/features/паспорт) один на проект, задачный слой docs/tasks/<слаг>/ — мини-пайп на задачу (свой STATE.md, ТЗ-дельта, свой гейт и DONE); 1 задача трекера = 1 пайп = 1 статус. (3) Сериализация внутри проекта (один пишущий на репо), параллель между проектами; непересекающиеся задачи одного проекта — опционально worktree по ADR.
Архитектура (выбор зафиксирован по волне A, отчёт docs/research/RS-01): слой 1 очередь = beads/bd (атомарный claim, ready-семантика, гейт = блокирующая задача на человеке); слой 2 диспетчер пишем сами (project-mutex, допуск max_workers = min(окно ccusage blocks, RAM/5GB→3-4 инстанса, число проектов), back-pressure по детерминированным строкам лимитов CLI (session/weekly/Opus), watchdog: heartbeat из PostToolUse-хука, петля/зависание → kill + СВЕЖИЙ рестарт от STATE.md — не resume); слой 3 гейт/уведомления = AskUserQuestion ждёт в терминале + Happy (push на телефон) / cmux за компом.
Ключевые ограничения из ресерча: OAuth-токен в Keychain → диспетчер = user LaunchAgent в GUI-сессии (не cron/LaunchDaemon), --bare не использовать; у батч-агентов нет прод-кредов; reference-тесты read-only вне worktree (reward hacking); агент-teams в воркерах не использовать (~7× токенов); все сессии тратят один пул лимитов; размер автономного куска — 80%-горизонт ≈ 27 мин (кап майлстоуна конвейера уже совпадает).
Research-вход: docs/research/RS-01-wave-A-architecture.md (+ appendix), волна S уже интегрирована в ядро конвейера (коммит a8d2f04 в ~/.claude).
Ведение: фазы 1-2 спекулятивно свёрнуты к lite (ресерч уже выполнен волнами); режим ратифицируется пользователем на гейте ТЗ.

## Карта рисков / сомнения агентов
- **ПРОБА (критично): RAM Mac = 8 GB** (свободно единицы сотен MB днём) → max_workers = 1 (иногда 2 на лёгких задачах ночью). Параллель между проектами на этой машине фактически недоступна по памяти — контур проектируется как ПОСЛЕДОВАТЕЛЬНАЯ автономная смена с очередью; project-mutex вырождается в «один активный воркер». В ТЗ — честно; опции расширения (RAM/облачные routines) — P2.
- Reward hacking в безнадзорном режиме: промпт-запреты бесполезны (METR 80→70%) → только внешние стены (read-only оракулы, deny-rules, гейт на «аномально быстрый зелёный»).
- Галлюцинация успеха (инцидент Gemini CLI) → пост-условия фаз проверяются фактами снаружи (git status/артефакты), не самоотчётом.
- claude agents/--bg — research preview: API может меняться; проба исполнимости обязана проверить текущую версию CLI.
- ccusage парсит нестабильный внутренний JSONL → нужен фолбэк (OTel/реактивный парсинг строк лимита).
- Keychain вне GUI-сессии недоступен → LaunchAgent Aqua; проверить на пробе.
