# RS-00 — Брейншторм: автономный таск-контур (12 углов, серийно)

Цель: утром накидал ~10 задач по проектам — Mac сам довёл их конвейером до конца без потери качества.

## Углы → инсайты → следствия для ТЗ

1. **Утренний оператор (завсегдатай)**. Если постановка 10 задач занимает >5 минут или требует вспоминать синтаксис — контур умрёт от трения на входе, а не от багов.
   → Следствие: однострочная постановка (`orc add <проект> "<задача>"` поверх bd) — первоклассная P0-фича, не придаток. Постановка пачкой из одного текстового блока.

2. **Вернувшийся вечером (турист по результатам)**. Ценность смены реализуется в момент «что случилось?». 10 открытых окон терминала — это НЕ ответ; разгребание = обесценивание всей автоматизации.
   → Следствие: команда-дайджест `orc status`: DONE / BETA / ждут гейта / упали, по строке на задачу, с путями к диффам и отчётам. Один взгляд — полная картина.

3. **Скептик «зачем это»**. Чем лучше, чем открыть 3 терминала руками? Если выигрыш не измерен — контур станет заброшенной игрушкой.
   → Следствие: метрики успеха в ТЗ: время «накидал → первый гейт готов к ответу», задач доведено/сутки, доля задач без ручного вмешательства (кроме гейтов).

4. **Юзер в худших условиях**. Ночью: Mac уснул, перезагрузился от обновления, claude обновился и сменил вывод, сеть отвалилась на 2 часа.
   → Следствие: восстановление после ребута/сна = P0 (LaunchAgent + caffeinate + резюм задач по их STATE.md); деградация «сеть упала» = парковка с ретраем, не фейл задачи.

5. **Агент как потребитель промпта**. Инстанс просыпается в терминале с одним стартовым промптом — этот промпт и есть API контура. Кривой промпт = все задачи стартуют вкось.
   → Следствие: шаблон стартового промпта (задача + workspace docs/tasks/<слаг>/ + ссылка на продуктовый слой + «вызови pipeline») — артефакт дизайна, проверяется consumer-тестом.

6. **Владелец репозиториев**. Что агентам можно БЕЗ спроса: коммиты в task-ветки? push в main? создать папку нового проекта где?
   → Следствие: секция «Полномочия» ТЗ с явным списком; deny-rules (push -f, main без ревью); новые проекты — только в согласованной директории.

7. **Безопасник**. Безнадзорная сессия читает веб → prompt injection; доступ к ~/.ssh, .env других проектов; эксфильтрация.
   → Следствие: sandbox с сетевым allowlist и denyRead чужих директорий = гейт качества (проверяемый); прод-креды в env воркеров отсутствуют по построению.

8. **Бухгалтер лимитов**. Один пул на всё; «лимиты недоиспользуются» сегодня ≠ всегда; смена из 10 задач может съесть недельный кап Opus.
   → Следствие: диспетчер логирует расход per-задача; дайджест показывает «смена съела X% окна / Y% недельного»; политика деградации Opus→Sonnet — явная настройка.

9. **Конкурент (Bernstein/Emdash/Conductor)**. Их сила — UI и generic-оркестрация; повторять это — проигрыш. Наша уникальность — нативная стыковка с конвейером pipeline (гейты качества, STATE.md-резюм, verify-фаза), которой нет ни у кого.
   → Следствие: не строить UI/kanban; строить тонкий клей вокруг конвейера. Всё, что можно взять готовым (bd, Happy, ccusage), — брать.

10. **Будущий я через 6 месяцев**. Модели меняются (горизонт удваивается ~131 день), число проектов растёт, форматы вывода CLI дрейфуют.
    → Следствие: калибровки (max_workers, размер смены, строки-детекторы лимитов) — в конфиг-файле, не в хардкоде; smoke-тест контура после обновления claude.

11. **Другой человек за этим Mac / семья**. Ночью вентилятор воет, батарея тает, утром кто-то сел — а там 5 живых сессий ИИ.
    → Следствие: kill switch «стоп всё» одной командой = P0; «тихий режим» (кап воркеров, только от сети); терминалы подписаны именами задач.

12. **Доступность/читаемость**. Дайджест и push-уведомления читаются на бегу с телефона: plain text, без таблиц-простыней, важное первой строкой.
    → Следствие: формат уведомления: «<проект>/<задача>: ЖДЁТ ГЕЙТА — вопрос одной строкой»; формат дайджеста — grep-able plain text.

## Синтез (топ-следствия для ТЗ)
- P0-скелет: постановка одной строкой → диспетчер → терминал с claude → гейт-парковка → дайджест → kill switch. Восстановление после ребута — P0.
- Уникальность = стыковка с pipeline (гейты, STATE.md, verify), не UI. Готовое — брать.
- Полномочия и безопасность (deny, sandbox, креды) — секции ТЗ, проверяемые гейтами.
- Все калибровки — конфиг; smoke после обновлений claude.
