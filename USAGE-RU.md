# orc — как пользоваться (шпаргалка)

`orc` — берёт твои задачи из очереди, сам поднимает Claude Code в терминалах,
гонит каждую через конвейер `pipeline` до конца и утром показывает «газету».

Все команды запускать из `~/orchestrator` (или добавь `bin/` в PATH).

---

## 1. Настроить один раз

```bash
cd ~/orchestrator
bin/orc init      # создаёт очередь (хаб ~/.orc)
bin/orc setup     # чтобы окна воркеров закрывались сами (правит профиль Terminal)
```

## 2. Накидать задачи

Проект — это папка с `git` (если нет — сделай `git init` внутри).

```bash
bin/orc add ~/Desktop/my-app "добавь тёмную тему на страницу настроек"
bin/orc add ~/Desktop/my-app "напиши тесты для auth" -p 0        # -p 0 = срочно
bin/orc add ~/Desktop/other  "почини баг с датами в отчёте"
```

Пачкой (вставить много строк сразу):
```bash
bin/orc add --batch
# затем вводишь строки вида:  проект: текст задачи   (Ctrl-D в конце)
```

Задача, где нужно ТВОЁ решение — помечай `--gate` (она дождётся тебя,
пришлёт macOS-уведомление, а остальные пройдут вперёд):
```bash
bin/orc add ~/Desktop/my-app "выбери редизайн главной" --gate \
  --scope "3 варианта лендинга" --cost "переделка = полдня"
```

Задача без интернета (максимум изоляции — воркеру отрежут сеть):
```bash
bin/orc add ~/Desktop/my-app "локальный рефакторинг модуля X" --offline
```

## 3. Запустить смену

```bash
bin/orc start           # поднимает claude по задачам, гонит очередь
bin/orc start --once    # поднять только один воркер и выйти
```
Откроются терминалы с настоящим Claude Code. Один проект — одна задача за раз
(не столкнутся). Гейтовые задачи идут последними, под твой ответ.

## 4. Смотреть, что происходит

```bash
bin/orc status              # живая картина: кто в работе, кто ждёт тебя
bin/orc status --newspaper  # газета: «N готово, M ждут тебя, K упало; потрачено ~X токенов»
```

## 5. Остановить / новая смена

```bash
bin/orc stop        # погасить всех воркеров (задачи вернутся в очередь)
bin/orc new-shift   # перед новой пачкой за день — сбросить старую смену
```

## Автозапуск (по желанию)

```bash
bin/orc install             # диспетчер сам поднимается в твоей сессии (LaunchAgent)
bin/orc install --uninstall # убрать
```

## Что важно помнить

- **Лимиты:** контур тратит квоту пока она есть. Уперся в реальный лимит Claude —
  паркует задачу до сброса окна и берёт следующую. Настраивать ничего не надо.
- **Безопасность:** воркеры НЕ могут писать вне своего проекта, удалять чужое,
  красть SSH-ключ или пушить — это заблокировано на уровне ядра. Но читать файлы
  вне проекта и ходить в сеть они могут (это нужно claude). Запускай задачи,
  которым доверяешь; для чувствительных — `--offline` или см. «Threat model» в `README.md`.
- **Гейты** приходят macOS-уведомлением. Ответил — задача поехала дальше.
- **Окна лимитов** сбрасываются каждые 5 часов; если квота есть — работает.

## Где что лежит

- Очередь и состояние смены: `~/.orc/` (`config.json`, `shift.json`)
- Настройки (пороги, denylist секретов, сеть): `~/.orc/config.json`
- Код контура: `~/orchestrator/src/orc/`
- Полное описание безопасности: `~/orchestrator/README.md`
